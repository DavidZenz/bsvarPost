# Technology Stack

**Analysis Date:** 2026-03-11

## Languages

**Primary:**
- R 4.1.0+ - Entire package implementation, Bayesian time series analysis

## Runtime

**Environment:**
- R 4.1.0 or later (as specified in `DESCRIPTION`)

**Package Manager:**
- CRAN (Comprehensive R Archive Network) - Primary distribution
- GitHub (DavidZenz/bsvarPost) - Development installation source

## Frameworks

**Core:**
- `bsvars` - Bayesian structural vector autoregressions estimation and computation
- `bsvarSIGNs` - Sign-restricted Bayesian SVAR models, includes native C++ routines

**Data Manipulation:**
- `tibble` - Tidy data frame construction and manipulation
- `rlang` - Tidy evaluation and non-standard evaluation helpers

**Visualization:**
- `ggplot2` - Grammar of graphics plotting system for all visualization

**Documentation & Build:**
- `roxygen2` 7.3.3 - Automatic documentation generation from code comments
- `knitr` - R markdown document compilation
- `rmarkdown` - Dynamic document generation
- `pkgdown` - Package website generation (Bootstrap 5, Flatly theme)

## Key Dependencies

**Critical:**
- `bsvars` (Imported) - Provides post-estimation operations like `compute_impulse_responses()`, `compute_variance_decompositions()`, `compute_structural_shocks()`, `compute_historical_decompositions()`, `forecast()`
- `bsvarSIGNs` (Imported) - Provides sign-restriction model classes (PosteriorBSVARSIGN) and native symbol exposure for restriction auditing
- `ggplot2` (Imported) - Essential for all plotting functionality in `R/plot-*.R` files
- `tibble` (Imported) - Used in all tidy extraction functions (`tidy_irf()`, `tidy_cdm()`, etc.)

**Utilities:**
- `methods` (Imported) - S3/S4 class system utilities
- `rlang` (Imported) - Used for tidy evaluation (`.data` pronoun) in plotting

**Optional (Suggested):**
- `tsibble` - Time series table conversion via `as_tsibble_post()` in `R/tsibble.R`
- `APRScenario` - Scenario analysis bridge via `as_apr_cond_forc()` in `R/aprscenario.R`
- `gt` - Table export to gt format via `as_gt()` in `R/reporting.R`
- `flextable` - Table export to flextable format via `as_flextable()` in `R/reporting.R`
- `tinytest` - Lightweight testing framework (used in `tests/tinytest.R`)
- `rmarkdown` / `knitr` - Required for building vignettes

## Configuration

**Environment:**
- No .env files - Configuration is via R function arguments and S3 method dispatch

**Build:**
- `DESCRIPTION` - Package metadata, dependencies, and Roxygen directives
- `NAMESPACE` - Generated by roxygen2, defines S3 methods and exports
- `_pkgdown.yml` - Website configuration (Bootstrap 5, Flatly theme, navbar structure)

**Documentation:**
- `.Rbuildignore` - Excludes non-essential files from package build
- `roxygen: list(markdown = TRUE)` in DESCRIPTION enables markdown in roxygen comments

## Platform Requirements

**Development:**
- R >= 4.1.0
- roxygen2 >= 7.3.3 for documentation generation
- Build tools for native package compilation (bsvars and bsvarSIGNs include C++ code)

**Production:**
- R >= 4.1.0
- bsvars and bsvarSIGNs packages installed (with native binaries compiled for target OS)
- ggplot2, tibble, rlang, methods (core dependencies)

**CI/CD:**
- GitHub Actions via `r-lib/actions` (setup-r, setup-r-dependencies, check-r-package)
- `ubuntu-latest` runner for R CMD check
- Pandoc required for pkgdown site generation

## Build & Test Infrastructure

**Local Pre-push:**
- Script: `tools/prepush-check.sh`
- Steps: roxygen documentation regeneration, source tarball build, R CMD check with `--as-cran` flag

**CI Pipeline:**
- **R-CMD-check** (`.github/workflows/R-CMD-check.yaml`):
  - Runs on: push to main, pull requests
  - Executes: R CMD check with `--as-cran --no-manual`
  - Uploads check logs and artifacts on failure
  - Uses `_R_CHECK_FORCE_SUGGESTS_=false` environment flag

- **pkgdown** (`.github/workflows/pkgdown.yaml`):
  - Runs on: push to main, manual workflow dispatch
  - Builds website to `docs/` folder
  - Deploys to GitHub Pages (`gh-pages` branch)
